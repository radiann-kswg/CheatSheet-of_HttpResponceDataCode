# 200 OK

> 引用: "The HTTP `200 OK` successful response status code indicates that a request has succeeded." — https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status/200

## 概要

- **分類**: 2xx（Success）
- **意味**: リクエストが成功したことを示します。メソッドにより「成功」の中身（GET は取得成功、POST は処理成功など）が変わります。
- **主な用途**: 取得/検索/処理など、成功時にボディを返す通常のレスポンス。

## 代表的なユースケース

- GET の成功（ボディにリソース表現を返す）
- POST の成功（ボディに結果メッセージや生成物を返す）

## 典型的な原因（よくある勘違い含む）

- 成功時でも常に同じ形式のボディが返ると思い込む（メソッドや API 設計に依存）
- 更新系（PUT/DELETE）で 200 を返しているが、実際は 201/204 の方が意図に合うことがある

## 関連ヘッダー / 関連ステータス

- 関連ステータス: `201 Created`, `204 No Content`, `304 Not Modified`

## 例

```http
HTTP/1.1 200 OK
Content-Type: application/json

{"message":"OK"}
```
